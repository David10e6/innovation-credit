# UI 修复与侧边栏布局优化 - 实现计划

## [x] 任务 1: 梳理各端已有功能清单
- **优先级**: P0
- **依赖**: 无
- **描述**:
  - 全面梳理当前项目中学生端、辅导员端、管理员端的所有已有功能
  - 按 "角色端（学生 / 辅导员 / 管理员）- 功能名称 - 功能路径 / 入口 - 核心操作逻辑" 的格式整理
  - 明确区分各端专属功能，无交叉混淆
  - 包含功能的完整操作流程
- **验收标准**: AC-3
- **测试要求**:
  - `human-judgment` TR-1.1: 功能清单是否完整覆盖所有已有功能
  - `human-judgment` TR-1.2: 功能清单格式是否规范，操作流程是否清晰
- **备注**: 此任务是后续修复的基础，需要详细梳理

## [x] 任务 2: 按已有功能定制各端专属侧边栏
- **优先级**: P0
- **依赖**: 任务 1
- **描述**:
  - 基于功能清单，为每个角色端定制专属侧边栏
  - 使用 Element Plus 的 ElMenu 组件，开启 Router 模式
  - 实现 "点击功能按钮→切换对应功能页面"，当前页面对应按钮自动高亮
  - 支持层级扩展（一级 / 二级菜单），预留 "新增功能按钮" 的接入位置
  - 侧边栏交互符合 Element Plus 官方规范
  - 侧边栏菜单配置采用 "可扩展数组格式"
- **验收标准**: AC-1, AC-3, AC-5
- **测试要求**:
  - `human-judgment` TR-2.1: 侧边栏是否仅显示对应角色的功能
  - `human-judgment` TR-2.2: 点击菜单是否能正确切换页面
  - `human-judgment` TR-2.3: 当前页面对应菜单是否自动高亮
  - `programmatic` TR-2.4: 新增菜单项是否能自动渲染到侧边栏
- **备注**: 侧边栏是核心导航组件，需要确保功能完整

## [x] 任务 3: 检查并适配已有功能的新界面风格
- **优先级**: P1
- **依赖**: 任务 2
- **描述**:
  - 逐一核查各端已有功能的操作界面
  - 原有功能的表单组件替换为 Element Plus 对应组件，保留原有校验规则
  - 原有功能的列表 / 表格替换为 Element Plus 的 ElTable 组件，保留原有筛选、分页、排序逻辑
  - 原有功能的弹窗 / 提示替换为 Element Plus 的 ElDialog/ElMessage 组件，保留原有触发逻辑
  - 所有功能操作的视觉样式与新界面风格统一
  - 输出核查清单，标注 "已适配 / 待适配" 功能
- **验收标准**: NFR-1
- **测试要求**:
  - `human-judgment` TR-3.1: 界面是否使用 Element Plus 组件
  - `human-judgment` TR-3.2: 视觉风格是否统一
  - `human-judgment` TR-3.3: 原有功能逻辑是否保持不变
- **备注**: 确保所有功能界面都适配新的 Element Plus 风格

## [x] 任务 4: 修复页面过窄、左右有空余的布局问题
- **优先级**: P0
- **依赖**: 任务 2
- **描述**:
  - 全局优化页面布局，消除浏览器全屏状态下页面左右的空余空间
  - 重置 html/body/#app 根容器样式：设置 width/height 为 100%、margin/padding 为 0
  - 后台布局容器（ElContainer）设置为 100vw（宽度）、100vh（高度）
  - 主内容区（ElMain）取消所有固定宽度 /max-width 限制，设置 width:100%、flex:1
  - 布局结构为 "左侧固定侧边栏 + 右侧全屏主内容区"
  - 适配不同浏览器分辨率
- **验收标准**: AC-2
- **测试要求**:
  - `human-judgment` TR-4.1: 浏览器全屏时页面是否无左右空余
  - `human-judgment` TR-4.2: 不同分辨率下布局是否正常
  - `human-judgment` TR-4.3: 主内容区是否自适应填充剩余空间
- **备注**: 布局问题是用户体验的重要因素，需要彻底解决

## [x] 任务 5: 确保登录页面独立渲染
- **优先级**: P0
- **依赖**: 无
- **描述**:
  - 确保登录页面独立渲染，不应用任何后台布局（侧边栏 / 顶部栏）
  - 仅保留原有登录功能
- **验收标准**: AC-4
- **测试要求**:
  - `human-judgment` TR-5.1: 登录页面是否独立渲染
  - `human-judgment` TR-5.2: 登录功能是否正常
- **备注**: 登录页面是用户的第一入口，需要确保独立且功能完整
