# 管理员端学分审核终审功能 - 产品需求文档

## Overview
- **Summary**: 为创新学分管理系统的管理员端增加学分审核终审功能，允许管理员对通过学院审核但未完成终审的学分申请进行最终审核，包括通过和驳回操作。
- **Purpose**: 完善学分审核流程，确保所有学分申请都经过学校层面的最终审核，保证学分认定的权威性和准确性。
- **Target Users**: 系统管理员（role: 3）

## Goals
- 实现管理员端学分终审页面，展示待终审的学分申请列表
- 实现审核通过功能，成功后增加学生创新学分
- 实现审核驳回功能，支持输入驳回原因
- 确保学生端能正确显示驳回状态、查看驳回原因并重新提交申请
- 保持与现有系统的风格一致性和交互流畅性

## Non-Goals (Out of Scope)
- 修改现有学生端的申请流程
- 修改学院审核的逻辑和流程
- 增加新的权限角色
- 实现批量审核功能

## Background & Context
- 系统已有的审核流程：学生提交申请 → 学院审核 → 学校终审
- 目前系统缺少学校终审环节的前端实现
- 后端API已提供相关接口：`getSchoolPending()`和`schoolAudit()`
- 系统使用Vue 3 + Element Plus构建，路由权限控制已完善

## Functional Requirements
- **FR-1**: 管理员可以访问学分终审页面，查看所有通过学院审核但未完成终审的学分申请
- **FR-2**: 管理员可以对申请进行审核通过操作，触发后端终审通过接口
- **FR-3**: 管理员可以对申请进行审核驳回操作，弹出输入驳回原因的弹窗
- **FR-4**: 管理员提交驳回原因后，触发后端终审驳回接口
- **FR-5**: 操作完成后，列表数据实时刷新，确保终审状态准确
- **FR-6**: 学生端能够显示驳回状态、查看驳回原因并重新提交申请

## Non-Functional Requirements
- **NFR-1**: 页面响应时间不超过2秒
- **NFR-2**: 操作反馈及时，成功/失败提示清晰
- **NFR-3**: 界面风格与现有管理员端页面保持一致
- **NFR-4**: 支持响应式布局，适配不同屏幕尺寸

## Constraints
- **Technical**: 基于现有Vue 3 + Element Plus技术栈
- **Business**: 遵循现有的审核流程和权限控制体系
- **Dependencies**: 依赖后端提供的`getSchoolPending()`和`schoolAudit()`接口

## Assumptions
- 后端API已实现并正常工作
- 现有路由权限控制机制有效
- 学生端已实现相应的状态展示和重新提交功能

## Acceptance Criteria

### AC-1: 终审页面访问权限
- **Given**: 用户已登录且具有管理员权限
- **When**: 用户访问终审页面路由
- **Then**: 页面正常加载，显示待终审列表
- **Verification**: `programmatic`

### AC-2: 待终审列表展示
- **Given**: 存在通过学院审核但未终审的申请
- **When**: 管理员进入终审页面
- **Then**: 列表正确显示申请数据，包括学生信息、申请内容等
- **Verification**: `human-judgment`

### AC-3: 审核通过功能
- **Given**: 管理员在终审页面
- **When**: 管理员点击某个申请的"通过"按钮
- **Then**: 系统调用后端接口，成功后列表刷新，该申请不再显示
- **Verification**: `programmatic`

### AC-4: 审核驳回功能
- **Given**: 管理员在终审页面
- **When**: 管理员点击某个申请的"驳回"按钮
- **Then**: 弹出输入驳回原因的弹窗
- **Verification**: `human-judgment`

### AC-5: 提交驳回原因
- **Given**: 管理员在驳回弹窗中输入原因
- **When**: 管理员点击"确认"按钮
- **Then**: 系统调用后端接口，成功后列表刷新，该申请不再显示
- **Verification**: `programmatic`

### AC-6: 学生端状态同步
- **Given**: 管理员完成终审操作
- **When**: 学生查看申请状态
- **Then**: 学生端正确显示终审结果和驳回原因（如果有）
- **Verification**: `human-judgment`

### AC-7: 重新提交功能
- **Given**: 申请被驳回
- **When**: 学生点击"重新提交"按钮
- **Then**: 申请状态重置，进入新的审核流程
- **Verification**: `programmatic`
