# UI 修复与侧边栏布局优化 - 产品需求文档

## Overview
- **Summary**: 对现有 Vue 项目进行 UI 修复和侧边栏布局优化，包括侧边栏功能按钮修复、界面大小调整、布局填充优化等，确保各角色端功能正常访问且界面美观。
- **Purpose**: 解决当前项目中侧边栏功能按钮无法切换页面、界面过窄、左右有空余空间等问题，提升用户体验。
- **Target Users**: 学生、辅导员、管理员等系统用户。

## Goals
- 修复侧边栏功能按钮，实现点击切换对应功能页面
- 优化界面布局，消除浏览器全屏状态下的左右空余空间
- 确保各角色端专属功能正确显示在侧边栏中
- 保持原有业务逻辑和功能代码不变，仅修改 UI 布局和样式
- 预留扩展性，支持后续新增功能的无缝接入

## Non-Goals (Out of Scope)
- 不修改任何业务逻辑代码
- 不增删任何功能
- 不修改权限校验规则
- 不改变接口调用方式

## Background & Context
- 当前项目基于 Vue 3 + Vite + Element Plus 构建
- 项目包含学生端、辅导员端、管理员端三个角色端口
- 存在侧边栏功能按钮无法切换页面、界面过窄等问题
- 需要保持原有功能不变，仅优化 UI 布局和交互

## Functional Requirements
- **FR-1**: 各角色端专属侧边栏菜单
  - 学生端：学院通知、学校通知、活动通知、学分申请、学分列表、申请记录、评分规则
  - 辅导员端：待审核列表、学生列表、活动管理、院级通知管理
  - 管理员端：用户管理、学生信息管理、学分统计、活动管理、校级通知管理
- **FR-2**: 侧边栏交互功能
  - 支持点击菜单切换对应功能页面
  - 当前页面对应菜单自动高亮
  - 支持菜单折叠/展开功能
  - 符合 Element Plus 官方交互规范
- **FR-3**: 界面布局优化
  - 消除浏览器全屏状态下的左右空余空间
  - 后台布局容器完全填充浏览器窗口
  - 主内容区自适应填充剩余空间
  - 适配不同浏览器分辨率
- **FR-4**: 登录页面独立渲染
  - 登录页面不应用后台布局
  - 仅保留原有登录功能

## Non-Functional Requirements
- **NFR-1**: 视觉一致性
  - 所有界面使用 Element Plus 组件和样式
  - 保持统一的视觉风格
- **NFR-2**: 扩展性
  - 侧边栏菜单配置采用可扩展数组格式
  - 后续新增功能可通过配置实现，无需修改核心代码
- **NFR-3**: 响应式
  - 适配不同屏幕尺寸
  - 小屏设备自动优化布局

## Constraints
- **Technical**: Vue 3 + Vite + Element Plus 技术栈
- **Business**: 保持原有功能不变
- **Dependencies**: Element Plus 组件库

## Assumptions
- 项目已正确安装 Element Plus 依赖
- 路由配置已正确设置
- 权限校验逻辑已完善

## Acceptance Criteria

### AC-1: 侧边栏功能按钮修复
- **Given**: 用户登录系统
- **When**: 点击侧边栏功能按钮
- **Then**: 系统应切换到对应功能页面，当前页面对应按钮自动高亮
- **Verification**: `human-judgment`

### AC-2: 界面布局优化
- **Given**: 浏览器全屏显示
- **When**: 访问系统后台
- **Then**: 页面应完全填充浏览器窗口，无左右空余空间
- **Verification**: `human-judgment`

### AC-3: 各角色端专属菜单
- **Given**: 不同角色用户登录系统
- **When**: 查看侧边栏菜单
- **Then**: 侧边栏应仅显示对应角色的功能菜单
- **Verification**: `human-judgment`

### AC-4: 登录页面独立渲染
- **Given**: 访问登录页面
- **When**: 查看页面布局
- **Then**: 登录页面应独立渲染，不显示侧边栏和顶部栏
- **Verification**: `human-judgment`

### AC-5: 扩展性验证
- **Given**: 需要新增功能
- **When**: 在菜单配置中添加新菜单项
- **Then**: 侧边栏应自动显示新功能按钮，点击可切换到对应页面
- **Verification**: `programmatic`
